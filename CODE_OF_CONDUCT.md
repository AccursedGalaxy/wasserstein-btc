# Code of conduct

This project follows the spirit of the
[Contributor Covenant v2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/),
adapted for a small research codebase.

## Our pledge

We — maintainers and contributors — pledge to make participation in
this project a productive, harassment-free experience for everyone
regardless of age, background, disability, ethnicity, gender identity,
experience level, nationality, race, religion, or sexual orientation.

## Standards

Behaviour that helps the project thrive:

- **Argue from evidence.** This is a research codebase. Disagreements
  about modelling choices are welcome — bring data, citations, or a
  failing test, not vibes.
- **Be precise.** "It's slow" / "it doesn't work" is not a bug report;
  see the issue templates in [`.github/ISSUE_TEMPLATE/`](.github/ISSUE_TEMPLATE/).
- **Be honest about negative findings.** The project documents its
  own failures (e.g. `WGeo-Hetero`); contributors should do the same.

Behaviour that is not acceptable:

- Personal attacks, harassment, or sexualised language.
- Publishing others' private information without explicit permission.
- Sustained disruption of discussions, issues, or PRs.

## Enforcement

The maintainer (Robin Bohrer, `robinbohrer7@gmail.com`) is responsible
for clarifying and enforcing these standards. Reports are confidential.
Possible responses include: a private warning, a public correction, a
temporary block, or a permanent ban from the project.

## Scope

This code of conduct applies in all project spaces — issues, PRs,
discussions, and any public spaces where an individual is representing
the project.
